01 — The Problem

Why hospitals get ICD coding wrong

Doctors write short, incomplete, ambiguous notes. ICD-10 demands extreme specificity. The gap between them costs billions.

Current reality
Naive coding systems
  • Take text at face value, no reasoning
  • Ignore ambiguity and missing information
  • Output one code with no explanation
  • Cannot handle negation ("no evidence of...")
  • Black box — no trust from clinicians
  • Wrong codes → claim denials + audits
ClinCode approach
Uncertainty-first reasoning
  • Detects what is missing before coding
  • Flags ambiguity and asks clarifying questions
  • Outputs ranked codes with calibrated confidence
  • Handles negation — never codes ruled-out conditions
  • Full explanation for every decision made
  • Feedback loop — improves with every correction
02 — How It Works

Six-stage reasoning pipeline

Each stage builds on the last. Uncertainty propagates forward. Nothing is discarded.

01
Preprocessing
Normalize + expand
Expand medical abbreviations, strip noise, tokenize. DM → Diabetes Mellitus. HTN → Hypertension. Every ambiguous shorthand resolved before analysis begins.
DM → Diabetes Mellitus  ·  CKD → Chronic Kidney Disease  ·  HTN → Hypertension
02
Medical NER
Extract entities
BioBERT/ClinicalBERT identifies diseases, symptoms, modifiers, and negated conditions. Each entity tagged with type and confidence score.
DISEASE: Type 2 Diabetes · CKD stage 3  ·  SYMPTOM: Proteinuria  ·  NEGATED: Retinopathy
03
Relationship Modeling
Understand context
Dependency parsing detects cause-effect relationships between entities. This is where most systems fail — ignoring that diabetes caused the kidney disease changes which code is correct.
Type 2 Diabetes → complication → CKD  ·  CKD → manifests as → Proteinuria
04
Ambiguity Detection — Core USP
Find what's missing
Rule-based checks scan for missing attributes. Diabetes without a type. MI without initial/subsequent. Asthma without severity. Each gap is flagged with a clarifying question instead of a silent guess.
FLAG: diabetes_type_unknown  ·  FLAG: ckd_cause_unclear  ·  RESOLVED: no_retinopathy (negation scoped)
05
ICD-10 Inference
Map to codes
SapBERT embeddings + FAISS similarity search across 70,000+ ICD codes. Outputs ranked candidates with calibrated confidence. Exploits ICD hierarchy — partial predictions are valid and useful.
E11.65 → 0.87  ·  N18.3 → 0.91  ·  E11.69 → 0.22 (alt)
06
Explainability
Show your reasoning
Every code is traced back to the exact text span that triggered it. Clinicians see which words drove each decision. Override buttons feed corrections back into the model training loop.
E11.65 selected because: "type 2 diabetes" + "proteinuria" + "CKD" co-occurrence pattern
